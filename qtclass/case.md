# 教学案例：Vibe Coding实践课

这是一篇量潮课堂的客户案例，打算给我们的 VIP 学员 Jerry 作为实践课作业，也作为未来制作新教学材料的标准案例。

## 课程简介

这是我们产教融合课程模式下的基于真实生产级项目的实践课。在这种课程模式下，我们直接拿出在做的生产级项目作为教学素材，拿出我们带实习生的培训模式带学生。我们的预期是帮助学生在三到六个月左右可以从零开始到可以成为合格的校招候选人。

实践课相比于理论课的主要区别是学生动手实践为主、老师评分和指导为主。我们也搬出了内部培训、绩效与薪酬的人力资源工作体系作为实践课的教学、评分与激励机制。

由于我们拿出的是真实的生产级开源项目，因此我们也会按照我们制定的开源激励政策发放激励。对于有效的交付，我们会返还现金或现金等价物（如代金券等）。满足实训生（一个月）或实习生（三个月）工作量标准也会按照规定发放实训和实习证明。具体政策以量潮课堂章程、VIP 客户具体章程、客户合同等正式文件为准。

## 业务背景简介

回顾上面的内容，实际上行文中隐含了大量标准数字资产及配套的工作流程。

开篇提到的“客户案例”是企业宣传册的一部分。企业宣传册是我们专门用来给客户宣传我们花多少钱能办多少事的数字资产。这篇文章的作用就在于告诉潜在的付费学员，我们的 Vibe Coding 实践课是什么样的。

“标准案例”是标准工作案例的简称。工作案例指的是我们用来指导内外部工作的案例资料。这些案例承担了普通法系的“判例”的功能，是具备约束力的文件。

标准工作案例是承担了“宪法判例”地位的最重要的案例。这里隐含了一条创始人特权——创始人的工作产出天然具备标准工作案例地位。

“章程”是工作章程的简称。工作章程可以理解为一类标准化合同。这类合同主要是约束参与各方的权责，通常是员工、公司、客户等多方共同遵守，相比于合同更公开透明和灵活变动。工作章程是公司的成文宪法的组成部分，是公司最重要的数字资产之一。

“实践课”和“理论课”是我们对标浙理计算机系的课程定义重组的定义，原本叫“训练营”和“基础课”。旧的概念依然会在宣传资料中使用，不过在正式协作时默认遵循浙理计算机系的定义。这样做的目的是为了充分使用优秀高校现有的教育工作体系，也是为了校企合作减少沟通摩擦。

以实践课为例，我们要给学生交付的资料包括：和本服务相关的工作章程、专门一对一签署的章程、合同、报价单等公司级标准资产，课上教学记录、课程任务书、作业批改结果等业务级标准资产。从这段描述我们可以看到，如果想要精确地使用数字资产描述整个业务交付产出，实际上是需要非常精细和复杂的。
